## Tooling

The IBM 2x Grid system is fully responsive and can be used to build diverse layouts thanks to a sixteen column system, five default breakpoints, Sass variables and mixins, and dozens of predefined classes.

Our grid is built with Flexbox and closely mirrors Bootstrap's grid in that it uses containers, rows and columns to layout and align content. Below is an in-depth look at how the grid comes together.

If you’re new to Flexbox, read this [CSS Tricks Flexbox guide](https://css-tricks.com/snippets/css/a-guide-to-flexbox/) for background, terminology, guidelines, and code snippets.

## Getting started

We provide multiple ways for you to use the grid for the Carbon Design System. By default, the grid code is included when you use `carbon-components`. To get started, you'll need to install this package using `npm`:

```bash
npm install carbon-components --save
```

Or, if you prefer Yarn:

```bash
yarn add carbon-components
```

You can include the grid in one of two ways:

```scss
// If you're including all the styles for Carbon
@import 'carbon-components/scss/globals/scss/styles.scss';

// If you're wanting to import the grid
@import 'carbon-components/scss/globals/grid/grid';
```

If you're hoping to use the 16 column variant of the grid, we recommend enabling the `grid-columns-16` feature flag before including any imports in your Sass files. For example:

```scss
$feature-flags: (
  grid-columns-16: true,
);

// If you're including all the styles for Carbon
@import 'carbon-components/scss/globals/scss/styles.scss';

// If you're wanting to import the grid
@import 'carbon-components/scss/globals/grid/grid';
```

If you're hoping to use the grid without including `carbon-components`, we offer the `@carbon/grid` package. You can install this with `npm`:

```bash
npm install @carbon/grid --save
```

Or, if you prefer Yarn:

```bash
yarn add @carbon/grid
```

You can then include the 16 column grid by doing the following in your Sass files:

```scss
@import '@carbon/grid/scss/grid';
```

#### Container

`bx--grid` defines the overall grid context at the top level and applies the proper margin and our default max-width (1584px.) You can also remove this max width, by using a grid modifier class. See the full width example below.

#### Rows

`bx--row` defines a row of items inside the grid — think of rows as wrappers for columns. Each column has horizontal padding (aka a gutter) that defines the space between them. You can negate this padding at the row level with negative margins, allowing the content within your columns to be aligned along the left side.

#### Columns

`bx--col` defines individual columns inside a row. Use as many columns as you’d like out of the possible 16 per row. With Flexbox columns widths are set in percentages relative to their parent element so columns without a spec’d width will automatically be equal. For example, four instances of .col-sm (or bx-col?) will will give you four equal width columns — each 25% of the parent from the small breakpoint on up. See the auto-layout columns for more on this.

### Responsive options

| Breakpoints     | Class prefix | # of cols | Gutter width\* |
| --------------- | ------------ | --------- | -------------- |
| Small (<320)    | `.col-sm-`   | 4         | 32px           |
| Medium (≥675)   | `.col-md-`   | 8         | 32px           |
| Large (≥1056)   | `.col-lg-`   | 16        | 32px           |
| X-Large (≥1312) | `.col-xl-`   | 16        | 32px           |
| Max (≥1584)     | `.col-max-`  | 16        | 32px           |

<Caption>
  * Gutter width is 32px — with 16px on each side of a column — at every
  breakpoint.

## Grid modes

In all, there are 3 different gutter scenarios in our system to allow for different types of content spacing: Comfortable, Cozy and Condensed. The gutters in between columns have a fixed width and do not scale.

The primary focus of these gutter scenarios is to enable typographic alignment. Both in editorial scenarios and in product scenarios with components. Containers can “hang” into the gutter but for the most part, type within or without containers stays on the 32px gutter scenario in the interest of typographic alignment.

### The comfortable grid

Our default grid is 16 columns and has 32px gutters. Unless your design specifies something different, this is the grid you should go with — it’s also arguably the easiest to use. Note, here, the container edge does not extend into the gutter, so the type within the container gets 16px top and right padding. This is the only scenario in which the type within a container does not align with the type outside of the container.

Rows that are adjacent and are comfortably arranged will have 32px of margin between them to match gutter. Use vertical helpers here.
